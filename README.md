# 译幕 YiMu

**AI 驱动的 SRT 字幕翻译工作台，翻译质量达到真人专业级别。**

> 目前仅针对 **英语 → 中文** 翻译进行了完整测试，其他语言对尚未验证，欢迎自行尝试并反馈。

---

## 为什么翻译质量能达到专业级别？

普通的 AI 翻译工具只是把原文丢给模型，逐句翻译就完事了。译幕不一样——它模拟了专业译者的完整工作流：

1. **先理解内容**：翻译前先分析视频领域和专业术语，生成术语表，确保全片术语一致
2. **再精准翻译**：术语表注入每一批翻译请求，避免同一个词前后翻译不同
3. **然后自动校对**：翻译完成后自动检测漏翻、短句等质量问题
4. **最后智能修复**：AI 判断问题字幕应该向上合并、向下合并还是跳过，软件自动执行修复

整个过程一键完成，无需手动干预。

## 功能特性

- **SRT 字幕上传** — 直接导入已有的 `.srt` 字幕文件
- **音视频转录** — 上传音视频文件，由本地 Whisper 模型自动生成字幕
- **短句合并** — 自动合并语音识别产生的碎片字幕，支持所有语言
- **AI 语音识别矫正** — 找出语音转文字中的错别字和误识别词（如 "deep sea" → "DeepSeek"）
- **一键翻译** — 内容分析 → 并行翻译 → 质量检测 → AI 智能修复，全自动流水线
- **并行翻译** — 多批次并发请求，大幅提升翻译速度
- **双语字幕导出** — 支持中英上下位置调换，预览后下载 SRT 文件
- **字幕编辑器** — 浮窗编辑器，支持搜索替换，Ctrl+F 快速打开

## 快速开始

### 环境要求

- Windows 10/11
- Python 3.9+
- 需要 AI 模型的 API Key（支持 OpenAI 兼容接口）

### 安装

```bash
# 1. 克隆项目
git clone https://github.com/mikuleader/srt-translator.git
cd srt-translator

# 2. 创建虚拟环境并安装依赖
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

### 启动

双击 `启动工作台.bat`，或手动运行：

```bash
python app.py
```

浏览器访问 `http://localhost:9999` 即可使用。

### 配置 AI 模型

首次使用需要在界面右上角 **⚙ 软件设置** 中配置两个模型：

| 模型 | 用途 | 已测试 |
|------|------|--------|
| 分析模型 | 字幕纠错分析 | Gemini 2.5 Flash / Pro |
| 翻译模型 | 字幕翻译 | GPT-4o / GPT-5.1 |

两个模型可以配置不同的 API Key 和 Base URL，也可以用同一个。其他模型未测试，欢迎自行尝试。

## 工作流程

```
导入文件 → 短句合并(可选) → AI 矫正 & 翻译 → 导出
```

**推荐使用「一键翻译」按钮**，自动完成以下四个阶段：

```
阶段 1: 内容分析  → AI 识别领域 + 生成术语表
阶段 2: 并行翻译  → 术语表注入，多批并发
阶段 3: 质量检测  → 检测漏翻和短句问题
阶段 4: AI 修复   → 智能合并问题字幕
```

## 技术架构

- **前端**：原生 HTML/CSS/JS 单页应用，无框架依赖
- **后端**：Flask + Waitress（端口 9999）
- **AI 调用**：OpenAI 兼容接口（`/chat/completions`）
- **转录引擎**：Faster Whisper（CLI 优先，Python 库备选）
- **数据存储**：纯浏览器内存，无数据库，刷新即清空
