# 多语言支持改造清单

> 当前版本仅测试了英语→中文，以下是支持其他语言对需要修改的位置。

---

## 严重（会直接出错）

| 文件 | 大致行号 | 问题 | 改法 |
|------|---------|------|------|
| `app.py` | 255 | 翻译 Prompt 写死"翻译成中文" | 改为模板，接收 `target_language` 参数 |
| `app.py` | 179 | 术语表格式写死 `"en"→"zh"` | 改为 `source_lang` → `target_lang` 动态字段 |
| `app.py` | 445 | 质量检测用中文 Unicode 范围（`\u4e00-\u9fff`）计算字数 | 根据目标语言选择对应的字符范围和阈值 |
| `app.js` | 1222 | 位置调换逻辑硬编码 `isChinese()`，非中文目标语言完全失效 | 改为通用的"源语言/目标语言"识别 |
| `index.html` | 356, 372 | 调换选项写死 `en2cn` / `cn2en` | 改为动态生成选项 |

## 中等（影响准确性）

| 文件 | 大致行号 | 问题 | 改法 |
|------|---------|------|------|
| `app.py` | 262 | 翻译输出格式列名写死"中文译文" | 改为 `{target_language}译文` |
| `app.py` | 169 | 内容分析 Prompt 假设英文源 | 根据源语言动态调整 Prompt |
| `app.py` | 216-234 | 修复诊断 Prompt 中文写死 | 模板化 |
| `app.js` | 95, 103 | 前端只有 `isChinese()` 判断 | 增加 `isJapanese()`、`isKorean()`、`isLatin()` |
| `app.js` | 274 | 语言名称映射硬编码 | 改为可扩展的映射表 |

## 轻微（显示/提示不准确）

| 文件 | 大致行号 | 问题 | 改法 |
|------|---------|------|------|
| `app.py` | 453 | 错误消息写死"中文字数不足" | 改为"目标语言字数不足" |
| `index.html` | 265 | 说明文案写死"翻译成中文" | 根据配置动态显示 |
| `index.html` | 352 | 调换说明写死"中文和外文" | 改为"源语言和目标语言" |

---

## 前置工作

1. 在 `config.json` 中增加 `source_language` 和 `target_language` 配置项
2. 设置面板增加语言选择（或文本输入）
3. 后端所有 Prompt 改为接收语言参数的模板
4. 前端语言检测函数扩展为多语言

## 注意

- 行号基于 2026-04-01 的代码版本，后续修改可能导致行号偏移，以关键词定位为准
- 改动涉及前后端十几处，建议针对具体语言对逐个测试
